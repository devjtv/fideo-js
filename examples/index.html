One video API for every provider.

Fideo is a tiny, framework-free video UI that wraps HTML5, YouTube, Vimeo, and Wistia behind a single set of data attributes. Responsive sources, background loops, viewport triggers, and themable controls — without shipping a player runtime.

Quick start

Two lines of HTML. That's the player.

Drop in the CDN bundle, mark up a <video> or <iframe> with the right attributes, and Fideo auto-discovers it on load. Same markup pattern for every provider.

Install via CDNHTML
<!-- styles + library -->
<link rel="stylesheet"
      href="https://cdn.jsdelivr.net/npm/fideo-js/dist/fideo.css">
<script src="https://cdn.jsdelivr.net/npm/fideo-js/dist/fideo.global.js"></script>
Drop in a playerHTML
<iframe
  data-fideo
  src="https://youtu.be/dQw4w9WgXcQ"
  data-fideo-poster="/poster.jpg"
></iframe>

Background mode

Hero videos that just behave.

One attribute turns any provider into a silent, looping, container-filling background — even when the source is a YouTube or Vimeo iframe.

Cinematic, without the bloat.

data-fideo-background hides controls, mutes audio, autoplays, loops, and object-fit covers the container — across every supported provider.

Background mode in the wild

Eight clips. Eight providers and ratios. One attribute.

Every tile below is a Fideo player in background mode — autoplay, muted, looping, cover-filling its container. Mix HTML5, YouTube Shorts, and Vimeo at any aspect ratio.

Vertical / Shorts

Portrait videos? Same attribute.

Pass a /shorts/ URL and Fideo normalizes it to the embed format automatically. Set data-fideo-background-aspect-ratio="9:16" so the iframe covers a portrait container without letterboxing — perfect for mobile-first hero modules and social cards.

Viewport playback

Plays when seen. Pauses when not.

Tie playback to an IntersectionObserver with a single attribute. Great for autoplaying card grids without burning CPU off-screen.

Scroll into view to start

viewport="play-pause"

Pause-only on exit

viewport="pause"
